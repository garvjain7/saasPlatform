# Project Setup Guide

## Prerequisites

- **Node.js** (v18+)
- **Python** (v3.10+)
- **Git**
- **PostgreSQL** (v14+)
- **Ollama** (optional, for local AI functionality)

---

## Quick Start

### 1. Clone and Install Dependencies

```bash
# Install all dependencies (Backend + Frontend + Monorepo)
npm run install:all
```

### 2. Database Setup

Ensure PostgreSQL is running and create the database specified in your `.env`:

```sql
CREATE DATABASE newdatainsights;
```

You can optionally run the initialization script to seed the database:
```bash
cd backend-node
npm run init-db
```

### 3. Python ML Environment

```bash
# Activate your virtual environment (recommended)
python -m venv .venv

# Windows:
.venv\Scripts\activate
# Mac/Linux:
source .venv/bin/activate

# Install ML dependencies
pip install -r ml_engine/requirements.txt
```

### 4. Backend Configuration

```bash
cd backend-node
cp .env.example .env
```

Update `.env` with your PostgreSQL credentials. 

> [!NOTE]
> MongoDB and Redis are no longer required for the core functionality.

### 5. Running the Application

You can run both the frontend and backend simultaneously from the root:

```bash
npm run dev
```

Alternatively, run them separately:

```bash
# Backend (port 5000)
npm run dev:backend

# Frontend (port 5173 - default Vite port)
npm run dev:frontend
```

---

## Storage Structure

The application creates several directories automatically:
- `uploads/raw/`: Storage for user uploads.
- `uploads/cleaned/`: Storage for finalized datasets.
- `ml_engine/artifacts/`: JSON metadata generated by the analytics pipeline.

- **Frontend:** http://localhost:5173
- **Backend API:** http://localhost:5000
